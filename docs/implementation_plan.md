# '행복한 의사' 챗봇 백엔드 구현 계획서 (Implementation Plan)

이 문서는 이전 '해빛코치(Habit Coach)' 챗봇의 아키텍처(Node.js + Express)를 참고하여, '행복한 의사' 전용 챗봇 백엔드를 구축하기 위한 기술 스펙과 구현 단계를 정의합니다.

**단체 정체성**: '행복한 의사'는 의료 취약계층(노숙자, 다문화 가정, 외국인 노동자, 주민등록 말소자, 의료보험 체불자 등) 환자분들을 온라인에서 돌보기 위해 자원봉사 의사들이 운영하는 비영리 봉사 단체입니다. 일반인도 이용 가능하나, 도움을 받은 일반 이용자에게는 소액 기부를 안내하여 활동 지속성을 확보합니다.

## 1. 아키텍처 개요 및 기술 스택
*   **언어 및 프레임워크**: Node.js, Express.js (해빛코치와 동일하여 유지보수 용이)
*   **AI 모델 (LLM)**: OpenAI API (ChatGPT) 또는 Google Gemini API (해빛코치에서 사용한 `@google/generative-ai` 재활용 가능)
    *   **역할**: 환자의 예진 데이터를 입력받아 의학 용어가 포함된 SOAP 형태의 차트로 요약.
*   **데이터베이스 (DB)**: Firebase Firestore 또는 Supabase
    *   **역할**: 추후 홈페이지(happydoctors.net) 연동 및 단체 활동 증빙을 위해, 모든 상담 내역(환자 입력, AI 1차 답변, 전문의 2차 답변, F/U 경과)을 기록하고 영구 보존.
*   **의료진 알림 전파**: Telegram Bot API (무료이며 단톡방 알림 봇 연동이 매우 쉽고 빠름)
*   **프론트엔드 연동**: 카카오 i 오픈빌더 (버튼식 폼 입력 후 백엔드로 환자 데이터 Webhook 전송)
*   **오픈채팅 안내봇**: 안드로이드 메신저봇R (MessengerBot R)

## 2. 모듈 및 디렉터리 구조 (예상)
`d:\antigravity\happydoctor\backend\` 디렉터리를 생성하여 작업합니다.

*   `package.json`
*   `index.js` (메인 Express 서버)
*   `.env` (환경 변수: LLM API KEY, TELEGRAM_BOT_TOKEN 등)
*   `routes/`
    *   `kakaoWebhook.js` (카카오 오픈빌더에서 예진 완료 시 호출되는 엔드포인트)
    *   `messengerBot.js` (환자 오픈톡방 안내용 + 의료진 단톡방 알림 발송용)
*   `services/`
    *   `llmService.js` (Gemini 프롬프트 관리 및 요약 로직)
    *   `notifyService.js` (의료진 단톡방으로 전송될 메세지 포맷 구현 및 큐 관리)
    *   `followUpService.js` (일정 시간 뒤 상태 점검 카카오 푸시 발송 스케줄러)

## 3. 핵심 엔드포인트 API 설계

### 🎯 API 1: 카카오 챗봇 메시지/예진 웹훅 (`POST /api/kakao/webhook`)
*   **동작 방식 (자율 해결 로직 적용)**: 
    1. 환자가 카카오 채널에서 문진(버튼/텍스트)을 진행하여 서버로 데이터가 전송됨.
    2. 서버가 `llmService`를 호출하여 **Gemini 2.5 Flash**에게 두 가지를 판단하게 함.
        *   판단 A (경증): 홈케어로 충분하거나 일반적인 안내 시 -> **AI가 직접 작성한 다정한 코칭 답변(인턴 보듬 말투) 생성**
        *   판단 B (협진/Red Flag 등): 전문의 확인이 필요 시 -> **SOAP 차트 요약 생성 + 협진 필요 플래그(Flag) 활성화**
    3. 서버 처리 분기:
        *   판단 A: 환자에게 AI의 1차 답변을 즉시 반환하고, 내부 타이머(15분 등)를 맞춰 `followUpService`에 예약.
        *   판단 B: 내부 큐에 차트를 적재, 환자에게 대기 안내 반환. (경과에 따라 F/U 타이머 예약).
    4. **상담 기록 DB 적재 (Logging)**: 환자의 기본 정보, 예진 데이터, Gemini의 판단 액션(A or B), 생성된 차트 내역 전체를 DB 서비스 모듈(`dbService.js`)을 거쳐 저장소에 타임스탬프와 함께 Insert 함. 차후 웹사이트에 노출/관리될 핵심 자산.

### 🎯 API 2: 환자 추적 관찰 자동 발송 (Follow-up Scheduler)
*   **동작 방식**: 
    1. Node.js 내부 스케줄러(`setTimeout` 또는 `node-cron`)가 작동하여 특정 환자의 설정된 대기 시간(예: 15분)이 경과했는지 감시합니다.
    2. 시간이 되면, 카카오톡 챗봇 **이벤트 API(Event API)** 또는 알림톡을 호출하여 봇이 먼저 환자의 카톡방으로 "상태가 어떤가요?" 메시지를 발송(Push)합니다.
    3. 환자가 답장을 보내면 다시 웹훅으로 들어오고, 이전 차트와 현재 차트를 비교(NRS 점수 변화 등)하여 급격한 악화 시 즉시 전문의 큐(`ESCALATE`)로 재전송합니다. 

### 🎯 API 3: 메신저봇R HTTP POST 엔드포인트 (`POST /api/messengerbot`)
*   '해빛코치'의 `messengerbot_script.js`를 변형하여 사용.
*   오픈채팅방 방장이 켜둔 공기계(메신저봇R)에서 작동. "상담", "아파요" 또는 특정 호출어(`!상담`) 감지 시, 
    **"안녕하세요! 진료 상담은 '행복한 의사 챗봇 채널'에서 1:1로 진행됩니다. [채널 링크]"** 를 자동 출력하도록 세팅.

## 4. User Review Required (검토 필요 사항)
> [!IMPORTANT]
> **1. 의료진 알림 매체 픽스:**
> 카카오톡 단톡방에 봇을 띄우려면 카카오톡 비즈니스 메시지(알림톡/친구톡 등 과금 발생) 또는 카카오워크(기업용)를 써야 하므로 제약이 많습니다. **무료이면서 단톡방 알림용으로 가장 널리 쓰이는 "텔레그램(Telegram) 봇"**을 의료진 소통방으로 사용하는 것을 제안합니다. 동의하시나요?
>
> **2. LLM 모델 선택:**
> 질문/답변의 자연스러움은 OpenAPI(ChatGPT)가 뛰어나고, 기존 '해빛코치' 코드를 활용해 무료/저렴하게 구축하려면 Google Gemini가 좋습니다. 예진표 요약 용도이므로 **Gemini 2.5 Flash**를 사용하여 비용을 아끼는 방향도 훌륭합니다.

## 5. Deployment (클라우드 배포 및 유지)
*   **호스팅 서비스**: **Render (무료 티어 이용)**
    *   Github 저장소 연동을 통해 CI/CD 자동 배포 환경 구성.
*   **특이사항 (Sleep 방지)**: Render 무료 티어는 15분 이상 외부 요청이 없으면 서버가 유휴 상태(Sleep)로 진입하여 다음 요청 시 응답 지연(Cold Start)이 발생함.
    *   **해결책**: 백엔드 코드 내부(`index.js`)에 `setInterval`을 활용하여 **14분 간격으로 자기 자신의 건강 상태 체크 API(`/`)를 호출(Ping)하는 유지(Keep-Alive) 로직**을 탑재함. 이를 통해 서버가 항상 깨어있는 상태를 유지하게 함.

## 6. Verification Plan (검증 및 테스트 계획)
*   **로컬 및 배포 후 웹훅 연동 테스트**: 오픈빌더 형태의 JSON 페이로드를 `/api/kakao/webhook`에 쏘아 자율해결 및 협진 응답 분기 동작 확인 완료.
*   **추적 관찰(F/U) 테스트**: `/api/kakao/fu-reply`를 통해 악화 시 전문의 알람이 정상 작동하는 것 확인 완료.
*   **개발 단계 진행**: 이 계획서 승인 이후, 코드를 Github에 푸시하고 Render에 연결하여 실서버 URL 획득 후 카카오 오픈빌더 연동 진행.
